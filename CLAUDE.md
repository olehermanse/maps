# Lektern Dive Map

Interactive underwater wreck map built with Next.js, React, and Tailwind CSS 4.

## Architecture

Single-page app — all UI lives in `src/app/page.tsx`. No routing, no API.

## Key concepts

- **Photo markers** (`photoMarkers` array): Positioned on the map with `x`/`y` percentages. Each has a `label`, depth (`m`), `size`, and an `imgs` array of filenames.
- **Label markers** (`labelMarkers` array): Text-only, no photos, not clickable.
- **Thumbnails**: Pre-generated square crops in `public/thumbnails/`, used for map markers. Generated by `generate-thumbnails.sh` from `public/photos/`.
- **Fullscreen view**: Opens when clicking a photo marker. Shows full image from `public/photos/` with carousel navigation if multiple images.
- **Zoom/pan**: Custom React implementation using refs for performance. Scroll wheel zooms toward cursor, drag to pan, pinch-to-zoom on mobile. Browser zoom is disabled.

## Image pipeline

- Raw photos go in `public/raw-photos/` (gitignored)
- `resize.sh` converts raw photos to web-sized JPEGs in `public/photos/`
- `generate-thumbnails.sh` creates 256x256 square crops in `public/thumbnails/` (gitignored, generated at build time)
- Image filenames in the `imgs` arrays are just filenames — `/photos/` and `/thumbnails/` prefixes are added in code

## Important patterns

- Use `vw` units (not `rem` or `px`) for marker text, borders, and border-radius to keep sizing consistent across mobile and desktop
- Use plain `<img>` tags (not Next.js `<Image>`) for thumbnails and fullscreen photos to avoid unwanted optimization/downscaling
- Zoom/pan state uses both React state (for rendering) and refs (for event handler calculations) to avoid stale closures
- The `openFullscreen` wrapper keeps `fullscreenRef` in sync so event listeners can check if fullscreen is open

## Commands

- `npm run dev` — generate thumbnails + start dev server
- `npm run build` — production build
- `npm test` — run Playwright tests
- `./resize.sh` — resize raw photos (run manually)
- `./generate-thumbnails.sh` — generate thumbnails (run manually or via dev/build)

## Docker

`Dockerfile` installs `imagemagick imagemagick-jpeg` in the builder stage to generate thumbnails before `npm run build`.

## Testing

Every new feature, interaction, or button must have a Playwright test in `tests/app.spec.ts`. Run tests with `npm test`. The dev server can run independently — Playwright reuses it if already running (`reuseExistingServer: true`).

Make sure tests and eslint are run after every change to verify correctness.
